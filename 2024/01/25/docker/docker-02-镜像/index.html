Docker 镜像

1.什么镜像

Docker 镜像是由一层一层的文件系统层组成的，每一层都是只读（read-only）。当从 Docker 镜像创建容器实例时，会在容器会在最顶层挂载一个可读可写的文件系统层，这样就可以运行任何进程。

当 Docker 从镜像创建容器时，最初的读写层是空的。随着容器内部状态的变化（文件的改变），也会发生在读写层。例如，当我们想改变一个文件（修改文件内容），Docker 实际会从镜像层复制一个相同的文件，然后在新的文件上修改。镜像层的文件内容并没有发生变化（镜像是只读的）。

2.构建镜像

构建镜像有两种方式：docker commit 或在 Dockerfile。使用Dockerfile的方式更加推荐和常用，因为 Dockerfile 构建镜像更加直观和便于维护。

docker commit

docker commit 可类似理解成 git 的 commit，我们可以基于现有的容器来制作新的镜像。

-m 选项指定提交信息；-a 选项指定作者名

Dockerfile

使用 Dockerfile 文件来创建镜像，会把 Dockerfile 所在的目录上传至docker 守护进程中去构建镜像，所以对于对构建镜像无用一些文件或者目录，可以通过 .dockerignore 文件来忽视。

创建一个 Dockerfile 文件

docker执行这些命令大概流程如下：

  • docker 会从 ubuntu 镜像创建一个容器
  • 然后在容器中执行指令，让容器状态发生变化
  • docker 执行 docker commit 构建一个新的镜像
  • docker 会从新的镜像再创建一个容器
  • 然后执行下一条指令，一直重复上面过程，直到所有指令都被执行完成

docker 构建镜像是一层一层的构建的，如果在构建过程中，某一步出错了，之前构建的层也是可以使用的。

RUN 指令在 Shell 上执行这些命令，例如：安装 nginx 等。如果运行 RUN 指令在没有 shell 的平台，我们可以通过 exec 的格式来执行

从Dockerfile中构建镜像

使用 docker build 来构建镜像

构建缓存

docker 在构建镜像时，会把每一层都作为镜像缓存，方便下次构建时，加快构建速度。为了让缓存失效有两种方式：一种改变Dockerfile里面的指令；在构建时增加 —no-cache 选项

3.Dockerfile 指令

Dockerfile有许多指令，这里只学习几个常用的指令：CMD、ENTRYPOINT、ADD、COPY、ENV、WORKDIR

CMD

CMD 在容器启动时，运行其指定的命令，和 RUN 指令类似，RUN 指令在镜像构建中运行命令。Dockerfile 中只能有一个 CMD 指令，如果有多个 CMD 指令，最后一个才生效。docker run 传递的命令会覆盖 Dockerfile 里面 CMD 指令。

等同在 Dockerfile 增加以下命令

ENTRYPOINT

ENTRYPOINT 指令和 CMD 指令的功能类似。但是 ENTRYPOINT 指令并不容易被 docker run 覆盖，docker run 所指定任何参数都会被传递到 ENTRYPOINT 指令。还可以和 CMD 指令搭配实现可选参数配置。

例如，在 Dockerfile 使用 ENTRYPOINT 指令，然后通过 docker run 指定 -g “deamon off;” 来启动 Nginx。

-g “deamon off;” 参数会被传递到 ENTRYPOINT 指令中，变成 usr/sbin/nginx -g “daemon off;”

CMD 指令和 ENTRYPOINT 一起使用，实现默认配置

当 docker run 没有任何参数时，容器将以 usr/sbin/nginx -h 运行

WORKDIR

WORKDIR 指令是设置工作目录的，也就当前所在哪个目录下

ENV

ENV 设置变量，后面的任何指令都可以使用这些变量

ADD

ADD 指令可以添加在构建上下文中的文件或着目录到镜像里。tar 类型会自动进行解压，也可以指定 URL

COPY

COPY 指令和 ADD 指令功能差不多，COPY 就单纯复制文件或目录。
