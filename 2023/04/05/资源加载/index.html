Resources 资源加载

Mybatis 加载XML资源是由 Resources类来完成的，Resources 类其实是包装了类加载，所以完成资源的加载是由 ClassLoader 来实现的。

ClassLoader 的类别

Java内置三种类加载器:

  • application or system 类加载器
  • extension 类加载器
  • bootstrap 类加载器

Bootstrap 类加载器

Bootstrap 类加载器负责加载 $JAVA_HOME/jre/lib 目录下类，它是其他类加载器的父类加载器，Bootstrap 类加载器不是由Java实现的，所以我们获取不到Bootstrap类加载器。

Extension 类加载器

Extension 类加载器的父类加载器是 Boostrap，它负责加载 $JAVA_HOME/lib/ext 目录或者由 java.ext.dirs 系统变量指定目录下的类。

System 类加载器

System 类加载负责加载 classpath 下或者由 -classpath、-cp 命令行指定参数下加载类，它是父类加载器是 Extension。

ClassLoader 获取资源和 Class 获取资源区别

Class.getResource() 方法获取资源，最终也是通过 ClassLoader 来实现的。

如果资源的路径名是已 / 开头，那么表明资源路径名是绝对路径；否则就是相对路径(相对于 Class 的包的路径)。

Resources

Resources 类(类名是名词)是负责加载资源，实际加载资源都是由类加载类负责，所以我们可以通过Resouces 类来加载classpath下的资源，它提供一些列方法：

  • getResourceAsStream()
  • getResourceAsReader()
  • getResourceAsFile()

这里列举三个方法，主要是看一下它方法命名：动词 + 名词。以Stream/Reader/File的形式获取资源。
